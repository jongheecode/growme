# V1.5: 데스크톱 셸(상단 네비게이션 + 레이아웃) & 프로필 페이지

## 배경

V1(디자인 시스템)까지 적용됐지만 데스크톱 화면에서 콘텐츠가 좁은 중앙 컬럼에만 머물러 여백이 과하다는 피드백, 그리고 홈/히스토리 외에 프로필로 갈 방법이 없다는 피드백을 받았다. 이 문서는 상시 노출되는 상단 네비게이션, 데스크톱에서 더 넓게 쓰는 레이아웃, 신규 프로필 페이지(기본정보 + 계정 설정)의 범위를 정의한다.

프로필의 "훈장"(뱃지) 표시는 아직 구현되지 않은 V2(`2026-07-15-v2-engagement-features-design.md`)의 `Badge`/`UserBadge`에 의존한다 — V2가 구현되기 전까지는 프로필에 빈 상태("아직 획득한 훈장이 없어요")로 표시한다.

## 목표

- 홈/히스토리/프로필이 항상 보이는 상단 네비게이션
- 데스크톱(넓은 뷰포트)에서 여백을 콘텐츠로 채우는 레이아웃
- 프로필 페이지: 기본 정보, 한줄소개(bio) 편집, 누적 통계 요약, 계정 설정(비밀번호 변경, 회원탈퇴)

## 데이터 모델 변경

`User`에 `bio String?` 필드 추가 (닉네임 아래 표시될 한줄소개, 기본값 없음).

기존 관계(`Activity.user`, `Session.user`, `Growth.user`)에 `onDelete: Cascade`를 추가한다 — 현재 스키마엔 명시가 없어 회원탈퇴 시 FK 제약으로 실패한다.

## 백엔드 API

`src/routes/users.ts` 신규 (`/api/users`, requireAuth):
- `GET /api/users/me` → `{id, email, nickname, bio, createdAt}`
- `PATCH /api/users/me` `{nickname?, bio?}` → 각각 `isNonEmptyString`(제공된 경우) 검증 후 업데이트. `bio`는 최대 60자 제한(빈 문자열 허용 = 초기화).

`src/routes/auth.ts`에 추가:
- `POST /api/auth/change-password` `{currentPassword, newPassword}` requireAuth → 기존 `passwordHash`와 `bcrypt.compare`, 일치 시 새 해시로 업데이트. 불일치면 401.
- `DELETE /api/users/me`(`users.ts`에 추가) requireAuth → 확인 문구를 body로 받지 않고(프론트에서 확인 모달로 처리), 즉시 `prisma.user.delete`(cascade로 하위 레코드 자동 삭제).

## 프론트엔드

### Layout 셸 재구성 (`frontend/src/components/Layout.tsx`)
헤더 nav에 "프로필" 링크 추가(홈/히스토리/프로필/로그아웃). `main` 컨테이너를 현재 `px-6 py-10`(중앙 정렬 좁은 컬럼)에서 `max-w-6xl mx-auto w-full px-6 md:px-10 py-10`로 넓혀 데스크톱에서 콘텐츠 폭을 키운다. 각 페이지 내부의 `max-w-sm`/`max-w-4xl` 제약은 페이지별로 필요에 따라 유지하거나 넓힌다(HomePage는 이미 `max-w-4xl` 그리드라 셸 확장만으로 자연히 넓어짐).

### 신규 `frontend/src/pages/ProfilePage.tsx` (`/profile`)
- `GET /api/users/me`, `GET /api/growth/me`로 기본정보+누적 통계 로드
- 표시: 닉네임, 이메일, 가입일(`createdAt`을 `YYYY.MM.DD 가입`으로 포맷), bio(없으면 "한줄소개를 남겨보세요" placeholder)
- bio 편집: 인라인 편집 필드 + 저장 버튼, `PATCH /api/users/me`
- 통계 요약: 누적시간(`growth.currentGauge`), 현재 단계(`growth.stage`) — 기존 데이터 재사용, 새 API 없음
- 뱃지 섹션: V2 미구현 상태이므로 "아직 획득한 훈장이 없어요" 빈 상태만 표시(하드코딩 문구, API 호출 없음 — V2 구현 시 이 섹션만 교체)
- 계정 설정 섹션: 비밀번호 변경 폼(현재 비밀번호/새 비밀번호, `POST /api/auth/change-password`), 회원탈퇴 버튼 → 확인 모달("정말 탈퇴하시겠어요? 모든 기록이 삭제됩니다") → 확인 시 `DELETE /api/users/me` → 성공하면 로그아웃 처리 후 `/login`으로 이동
